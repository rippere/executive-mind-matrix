## How It Works

1. **Startup:**
   - Application creates TaskScheduler instance
   - Injects Notion AsyncClient via `set_notion_client()`
   - Scheduler registers cron job based on `command_center_refresh_interval`

2. **Execution (Every N Minutes):**
   - Scheduler triggers `refresh_command_center_metrics()`
   - Method creates FinalCommandCenter instance
   - Calls `update_metrics_only()` (existing method - no changes needed)
   - Logs success/failure and execution time
   - Records Prometheus metrics

3. **Error Handling:**
   - Errors are logged but don't crash the scheduler
   - Failed refreshes are tracked in Prometheus metrics
   - Scheduler continues running and retries on next cycle
